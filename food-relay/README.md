# 店舗間フードリレー 食材の貸借・精算（試作版 / たたき）

> 多店舗運営の「店舗間の食材の貸し借り」を、申請→他店が承認→実績化まで一気通貫で扱い、
> 食材の貸し借りを金額ベースで自動可視化し、月末にまとめて精算するシングルファイルWebアプリの **試作（たたき）** です。
> 姉妹アプリ「店舗間ヘルプ精算（ヘルプリレー）」と店舗マスタ・精算ロジックを揃えており、将来は人件費＋食材を1本で月末精算する前提です。
> 実店舗データは含まず、架空のサンプルデータで動作します。

🔗 **デモを開く / Live demo:** https://spicy-device.com/food-relay/

---

## これは何？
「店舗間で食材を融通し合ったときの貸し借り」を見える化・精算する叩き台です。
役割（本部／店長／店舗スタッフ）でログインを切り替えると、見える情報と操作が変わります（本番はログイン認証制）。

## 精算のコア（ヘルプリレーと共通）
- **店舗マスタ**：ヘルプリレーと同一（将来の人件費＋食材の合算精算に備える）
- **貸借マトリクス**：縦＝貸した店（貸し手）／横＝借りた店（借り手）、金額ベース
- **店舗別バランス**：貸した額 − 借りた額の差引
- **精算リスト（相殺後）**：店舗ペアごとに相殺した「誰が誰へいくら払うか」

## 食材固有のモデル
- **品目マスタ**：食材名・単位・標準単価（例：トマト/kg、牛肉/kg、醤油/本、生ビール/樽）
- **標準単価**は本番で **発注サイト（インフォマート等）の仕入単価を自動取込** する想定。デモでは「🔄 取込」ボタンで連携イメージを体験できます。
- **貸借記録**：日付・貸した店・借りた店・品目・数量・単価 → 金額（数量×単価）。CSV出力・月次フィルタ対応

## 申請フロー（備忘記録型）
1. **申請**：借りたい店が「品目・数量・希望受渡日」を入力し、声をかけたい店舗（複数）を選ぶ
2. **通知**：選ばれた店に申請が届く（通知バッジ）
3. **承認**：貸せる店が承認すると、自動で貸借記録（実績）に反映
4. 辞退・取消にも対応

## 役割と権限
- **本部 / オーナー**：全店舗の閲覧、品目マスタと標準単価の管理、申請の承認、精算管理
- **店長**：自店の食材申請を出す、他店からの申請を承認、記録・精算・品目の確認
- **店舗スタッフ**：自店の食材申請、貸借状況の確認
- ※食材単価は時給ほど秘匿性が高くない想定のため、金額は各役割に表示（マスタ・単価の編集は本部・店長のみ）

## 技術 / 構成
- `index.html` 1ファイル完結（Vanilla JS、外部ライブラリ非依存）
- データ保存: ブラウザの localStorage（閲覧者の端末内のみ。他の人の入力は共有されません）

## このデモについて
- 起動時に架空のサンプル（4店舗・品目8種・貸借記録・申請）を自動投入します。
- 役割の切替はヘッダー右上のプルダウンで行えます（デモ用の擬似ログイン）。
- 「店舗管理 → サンプルデータで初期化」でいつでも初期状態に戻せます。
- これは要件確認用の試作です。ログイン認証・サーバー保存・発注サイト連携・通知などは本開発で対応予定です。
