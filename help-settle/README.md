# 店舗間ヘルプ 申請・人件費精算（試作版 / たたき）

> 多店舗運営の「店舗間ヘルプ（応援勤務）」を、募集→応募→本部承認→実績化まで一気通貫で扱い、
> 人件費の貸し借りを自動で可視化・精算するシングルファイルWebアプリの **試作（たたき）** です。
> 実店舗データは含まず、架空のサンプルデータで動作します。

🔗 **デモを開く / Live demo:** https://spicy-device.com/help-settle/

---

## これは何？
「ジョブカンの簡易版」イメージの叩き台です。役割（本部／店長／店舗スタッフ）でログインを切り替えると、
見える情報と操作が変わります（本番はログイン認証制）。

## 役割と権限
- **本部 / オーナー**：全店舗の閲覧、募集のマッチング承認、時給・精算の管理
- **店長**：自店の募集を出す、自店スタッフのスキル編集、時給・自店実績の確認
- **店舗スタッフ**：他店募集への応募・自店の応援申請。**時給は表示されません**

## 申請フロー
1. **募集（申請）**：日付・時間帯・ポジション・必要レベルを指定。特定の人の**指名**も可
2. **応募**：他店スタッフが応募
3. **本部がマッチング・承認**：応募者から担当を確定
4. **実績化**：確定すると実績（人件費・店舗間の貸借）に自動反映

## スキル管理
- 各スタッフに「ポジション × レベル（研修中／可／Lv2／Lv3）」を保持
- 店長・本部が編集可。募集の必要レベル・指名・マッチング候補の絞り込みに使用

## 技術 / 構成
- `index.html` 1ファイル完結（Vanilla JS、外部ライブラリ非依存）
- データ保存: ブラウザの localStorage（閲覧者の端末内のみ。他の人の入力は共有されません）

## このデモについて
- 起動時に架空のサンプル（4店舗・8名・募集3件・実績）を自動投入します。
- 役割の切替はヘッダー右上のプルダウンで行えます（デモ用の擬似ログイン）。
- 「店舗管理 → サンプルデータで初期化」でいつでも初期状態に戻せます。
- これは要件確認用の試作です。ログイン認証・サーバー保存・通知・打刻連携などは本開発で対応予定です。
